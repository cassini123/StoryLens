## 4. HCI 设计原则 ↔ 系统实现

| HCI 原则 | StoryLens 实现 | 用户收益 |
|----------|-----------------|---------|
| **渐进披露** | 两轮提问（固定三维 → LLM 细化） | 降低认知负荷，避免信息过载 |
| **结构化选择** | A/B/C/D 卡片 + 文本 D | 快速决策 + 保留表达自由 |
| **门控操作** | LoRA 确认后才能生图 | 防止误触、明确风格意图 |
| **可选增强** | TT 专家非强制 | 新手低门槛，进阶可叠加 |
| **一致心智模型** | Agent 对话 ≡ HTML 三页流程 | 跨界面无学习成本 |
| **即时反馈** | 解析摘要 + KG 三视图 | 可见性原则，理解 AI 推理 |
| **可撤销/可改** | 历史记录 v2、专家可取消 | 用户控制感 |
| **错误容忍** | 生图四重回退 | 降级而非失败 |

## 5. 决策树：何时加载什么

```
用户意图
├── 仅要分镜产出？
│   ├── 是 → 本 Skill + 主 Skill #2
│   └── 否 → 专家完整 Skill（退出交集域）
├── 是否点名专家？
│   ├── 是 → H（当前模式）+ V（agents/*.md 节选）
│   └── 否 → 仅 H；关键词命中则自动 V
├── 是否产品场景？
│   ├── 是 → 建议 H=product-concept + V=louyongqi
│   └── 否 → 默认 H=film-storyboard
└── 是否在 Cursor 对话？
    ├── 是 → 遵循 Agent 执行步骤 1-9
    └── 否 → HTML appState 驱动
```
