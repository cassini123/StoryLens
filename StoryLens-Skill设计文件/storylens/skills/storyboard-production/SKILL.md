# 分镜制作 · 交集域 Skill

> **大前提**：StoryLens 的一切 Agent 行为，必须在「分镜制作领域」内发生。  
> 主 Skill 提供横向创作框架，TT 专家提供纵向方法论纵深；**有效输出 = 两者交集**，而非专家全库照搬。

## 一、双层坐标系

```
                    纵向 · TT 专家纵深
                         ↑
    辛向阳(交互/行为)     │     娄永琪(产品创新)
    李何槿(可持续)        │     柳喆俊(严肃游戏)
                         │
    ─────────────────────┼─────────────────────→  横向 · 主能力模式
                         │
              🎮 游戏设计  🎬 影视分镜  📐 产品概念  🎨 创作灵感
                         │
                         ▼
              ★ 交集域 = 分镜制作有效内容 ★
```

| 维度 | 代号 | 来源 | 职责 |
|------|------|------|------|
| **横向** | H | `SKILL_MODES[]` / 主 Skill Core Capabilities | 决定创作方向、KG 分支、引导提问维度 |
| **纵向** | V | `TT_EXPERTS[]` / `专家skill/` | 注入人格、方法论、评审视角到 parse/refine/对话 |
| **交集** | H ∩ V | 本 Skill | 专家只在分镜流程各阶段贡献**与当前场景相关的**方法论切片 |

## 二、框定范围（In / Out）

### ✅ In Scope — 分镜制作领域

- 文字场景 → LLM 解析 → 两轮引导提问 → 帧数确认 → LoRA 门控 → 分镜生成
- 镜头语言：景别、机位、运镜、节奏、情绪曲线、转场
- 每帧 KG：4–6 层维度可视化（SVG 环图 + D3 + 树形）
- 主能力模式下的领域化提问（MDA / OPC / 严肃游戏 / 画师风格）
- TT 专家在 parse / refine / AI 面板中的**可选**人格注入
- 输出：合并 PNG、单帧 PNG、JSON/CSV/PDF

### ❌ Out Scope — 不在本 Skill 处理

- 纯学术咨询、职业建议、无视觉产出需求（→ 路由到专家 Agent 完整 Skill）
- 跳过引导提问直接生图（硬性禁止）
- 专家子 Skill 全量加载（如 workshop-designer、critique 5 段式）而不经过分镜上下文
- 与分镜无关的产品战略、融资、组织管理

## 三、横向 × 纵向 交集矩阵

> 每个单元格 = 该主能力 + 该专家在分镜流程中的**有效协同点**。

| 主能力 ↓ \ 专家 → | 🌻 辛向阳 | 👓 娄永琪 | 🍐 李何槿 | 6 柳喆俊 |
|-------------------|----------|----------|----------|----------|
| 🎮 **游戏设计** | 交互行为在关卡中的视觉节拍；UI/UX 反馈镜头 | 游戏化产品创新 Pitch 分镜 | 可持续主题关卡叙事 | 严肃游戏教学场景分镜 |
| 🎬 **影视分镜** | 用户旅程 → 镜头调度映射 | 产品概念方案视觉叙事 | 循环经济/旧物改造故事板 | 教育场景/VR 体验分镜 |
| 📐 **产品概念** | 交互逻辑与任务流可视化 | **OPC / Pitch / CMF 主协同** | 可持续产品使用场景分镜 | 游戏化产品演示分镜 |
| 🎨 **创作灵感** | 同人/自媒体中的交互体验表达 | 社会创新视觉叙事 | 旧物改造/品牌故事分镜 | 游戏自媒体内容分镜 |

**典型最强组合**：`product-concept` + 娄永琪 → 产品框架 + 方案评审视角

## 四、专家介入点（流程内）

```
INPUT                    DRAFT                   RENDER
  │                        │                       │
  ├─ POST /parse ──────────┤                       │
  │   注入 V 人格            │                       │
  ├─ 第一轮 Q1-Q3 ─────────┤                       │
  ├─ POST /refine ─────────┤                       │
  │   注入 V 人格            │                       │
  ├─ LoRA 门控 ────────────┤                       │
  └─ POST /generate ───────┼─ KG 展开 ─────────────┤
                           │  H 模式 KG            │ 导出 PNG/JSON
                           └─ AI 面板对话 ─────────┘
                               H + V 前缀叠加
```

| 阶段 | 横向 H 贡献 | 纵向 V 贡献 | JSON Schema |
|------|------------|------------|-------------|
| 解析 | 按 activeSkillMode 选 KG 维度 | llm_proxy 读取 `agents/*.md` 节选 | 固定 |
| 第一轮提问 | 表达/场景/人物三维卡片 | 专家偏好的追问措辞 | 固定 |
| 第二轮提问 | 细化镜头/影调/构图 | 专家方法论切片 | 固定 |
| 生成 | 风格抽屉 + 帧结构 | genPrompt 专家语义 | 固定 |
| AI 对话 | 当前分镜真实数据 | prefix + style 语句 | — |

## 五、Agent 执行规则

1. **先确认交集**：用户请求是否属于分镜制作？否 → 路由专家完整 Skill 或主 Skill 其他 Capability
2. **加载本 Skill**：进入分镜流程时读取本文件 + `intersection-matrix.md`
3. **横向优先**：默认 `film-storyboard` 模式；用户切换主能力或 HTML 同步时跟随 H
4. **纵向可选**：用户点名专家 / 关键词命中 / HTML 手动选择 → 加载 `专家skill/{slug}/agents/*.md` 节选（非全量）
5. **硬性同步**：Agent 步骤必须与 `storylens-generator.html` 一致，不可跳步
6. **HCI 一致**：对话侧复现 A/B/C/D 卡片式提问，不一次性抛 10 个问题
