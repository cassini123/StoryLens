# 游戏设计核心框架

## 1. MDA 框架 (Mechanics-Dynamics-Aesthetics)

由 Hunicke, LeBlanc, Zubek 在 GDC 2004 提出。从三个层面分析游戏：

### Mechanics（机制层）— "游戏的规则是什么？"
- 游戏的基本操作和规则定义
- 玩家能做什么、不能做什么
- 资源系统、胜利条件、失败条件
- 例：马里奥的跳跃高度、敌人碰撞判定、金币收集规则

### Dynamics（动态层）— "规则运行时会发生什么？"
- 机制之间的交互产生的 emergent 行为
- 玖家的策略选择和博弈过程
- 难度曲线和学习曲线
- 例：马里奥中玩家利用"跳踩敌人"的机制发展出的连跳策略

### Aesthetics（美学层）— "玩家获得了什么情感体验？"
- 玩家的情绪反应和内心感受
- 8 种核心美学类型：
  1. **Sensation** — 感官享受（视觉/听觉冲击）
  2. **Fantasy** — 幻想/代入感（成为另一个人）
  3. **Narrative** — 叙事驱动（故事体验）
  4. **Challenge** — 挑战（克服障碍的成就感）
  5. **Fellowship** — 社交连接（合作/竞争）
  6. **Discovery** — 探索发现（学习新事物）
  7. **Expression** — 自我表达（创造/定制）
  8. **Submission** — 逃避现实（消磨时间）

**设计原则**：自上而下设计——先确定想要的 Aesthetics，再设计产生它的 Dynamics，最后定义实现这些 Dynamics 的 Mechanics。

## 2. 四象限设计法

将游戏拆解为四个维度进行独立分析：

| 象限 | 核心问题 | 关键考量 |
|------|---------|---------|
| **机制 (Mechanics)** | 怎么玩？ | 规则一致性、反馈清晰度、深度 vs 上手难度 |
| **叙事 (Story)** | 为什么玩？ | 世界观、角色动机、情节推进方式 |
| **美学 (Aesthetics)** | 看起来怎么样？ | 视觉风格、音效设计、UI/UX |
| **技术 (Technology)** | 用什么实现？ | 平台限制、性能优化、工具链 |

**应用场景**：适合在项目初期做"健康检查"，发现哪个维度被忽视。

## 3. 玩法核心循环 (Core Gameplay Loop)

每个成功的游戏都围绕一个或多个核心循环构建：

```
┌─────────────────────────────────────┐
│                                     │
│   动作 → 反馈 → 奖励 → 动机增强     │
│     ↑                           │  │
│     └───────────────────────────┘  │
│                                     │
└─────────────────────────────────────┘
```

**分析步骤**：
1. 找出玩家最频繁执行的动作序列（通常 3-5 步）
2. 分析每一步的反馈是否即时且有意义
3. 检查奖励是否能强化动机形成闭环
4. 评估循环的可变性和长期吸引力

**常见循环类型**：
- **采集循环**：探索 → 收集资源 → 升级装备 → 探索更远区域（RPG）
- **战斗循环**：遭遇 → 决策 → 执行 → 结果 → 成长（动作游戏）
- **建造循环**：规划 → 收集 → 建造 → 使用 → 新需求（模拟经营）

## 4. 心流理论在游戏中的应用 (Flow Theory)

Csikszentmihalyi 的心流模型：

```
挑战度 ↑
    │
高 │   焦虑区      心流通道      ────────
    │      ╲        ╱           ╱
    │       ╲      ╱           ╱
中 │        ╲    ╱           ╱
    │         ╲  ║          ║
    │          ╲║          ║         无聊区
低 │═══════════╪══════════╪═════════→ 技能度
             低              中          高
```

**设计启示**：
- 新手阶段：低挑战 + 低技能引导 = 不无聊不焦虑的入门区
- 进阶阶段：同步提升挑战和技能，保持玩家在心流通道
- 高手内容：高挑战 + 高技巧要求 = 极限心流
- **DDA（Dynamic Difficulty Adjustment）**：根据玩家表现实时调整难度

## 5. Bartle 玩家分类模型

四种基本玩家类型及其设计应对：

| 类型 | 核心驱动力 | 设计要点 |
|------|-----------|---------|
| **Achievers (成就型)** | 积累、升级、排行榜 | 明确进度条、成就系统、数值成长 |
| **Explorers (探索型)** | 发现秘密、理解世界 | 隐藏内容、彩蛋、世界 lore |
| **Socializers (社交型)** | 与他人互动 | 公会、交易、合作任务 |
| **Killers (竞争型)** | 影响其他玩家 | PvP、排行榜、声望系统 |

> 实际玩家通常是混合型。设计时应覆盖至少两种类型的诉求。
