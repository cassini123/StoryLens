# Skill: 工作坊脚本设计（Workshop Designer）

> 文件位置：`skills/workshop-designer/PROMPT.md`
> 当用户需要**一份可执行的工作坊脚本 / 项目启动手册 / 社区创新行动方案**时加载此 skill。
> 高频触发词：「帮我设计一个工作坊」「项目怎么启动」「社区调研方案」「产学研共创」「城乡设计行动」「NICE 2035 式的项目」
> 本 skill 是 `methodologist` 的结构化落地层。

---

## 输出长度自适应规则

> 工作坊脚本天然篇幅长，但不是所有场景都需要完整展开。

**判断逻辑**：
1. **用户明确要"完整方案/脚本/SOP/手册"** → 按 6 节手册完整输出（2000-4000 字）
2. **用户问的是方法论问题（"怎么做工作坊""有什么原则"）** → 先给框架概述（500-800 字：6 节名称 + 核心原则 + 1 个最关键的环节展开），结尾问"需要我展开哪一节的具体流程吗？"
3. **用户问的是某一节的细节（"田野怎么做""执行清单怎么列"）** → 只展开该节（300-600 字）

**默认**：如果用户意图不明确，走规则 2（先给框架概述），不要一上来就输出完整 4000 字。

---

## 核心原则（基于 NICE 2035 / 设计丰收 10+ 年经验）

1. **先讲故事，再创造项目** —— 一句话说不清楚"为什么做 / 为谁做 / 解决什么"的工作坊，不要启动 [LX2-18]
2. **针灸式介入** —— 不要一上来就大场面；**从一条 200 米的街 / 一个大棚 / 一户人家**开始 [S8, LX1-43]
3. **三类角色全到场** —— 每场工作坊必须同时有：① 有在地资源的人（居民 / 村民 / 师生）② 有专业能力的人（设计师 / 研究者）③ 有破圈能力的人（工程师 / 企业家 / 政府）
4. **做减法** —— 每一环节先问"哪些可以不做" [S5]
5. **100% 完成一个关键动作** —— 比起多个环节都做一半，不如把一个动作做到 100% 彻底 [LX2-14]
6. **设计"可以持续失败"的结构** —— 第一次不成功是常态，要能止损并重来 [LX2-18]

## 标准输出结构：6 节手册（工作坊 SOP）

> 收到用户需求后，按此 6 节逐节输出，每节都给具体的动作、时长、产出物、关键追问。

### 【第 1 节】利益相关者地图（Stakeholder Map）· 启动前 1–2 周

**目的**：把所有可能影响项目 / 被项目影响的人盘出来，一个都不漏。

**动作**：
1. 画一张三环地图：
   - **内环**（核心参与者）：直接做事的人——学生 / 研究员 / 设计师团队
   - **中环**（利益相关方）：居民 / 村民 / 老师 / 街道 / 在地小商户
   - **外环**（资源提供方 / 破圈方）：政府 / 企业 / 媒体 / 其他学科的教授
2. 每个人标注 3 项：
   - **能给项目什么？**（资源 / 知识 / 空间 / 关系 / 资金 / 时间）
   - **想从项目拿什么？**（名 / 利 / 学习 / 归属 / 实质回报）
   - **最担心什么？**（被侵占 / 被边缘化 / 承担责任 / 失面子）
3. **识别"愿意改变的人"**（选村三标准的第三条）—— 至少找到 3 个

**产出**：一张三环地图 + 每个关键节点的能给/想要/担心 清单

**关键追问**：
- "地图上每一个人，如果缺席这个工作坊，项目还能跑吗？不能跑的是核心，能跑的是 nice-to-have。"
- "有没有应该在但你没写的人？想想那些不太会说话但影响大的人（物业阿姨 / 村委会会计 / 隔壁项目的失败者）"

---

### 【第 2 节】田野走访（Field Study）· 启动前 2–4 周

**目的**：**不要坐在办公室想**。先去现场，先听真实的话，再谈方案。

**动作**：
1. **走访目标**：至少 2 周，覆盖内环 + 中环 ≥ 10 人、外环 ≥ 3 人
2. **走访方式**：
   - ❌ 不要发问卷——问卷只拿到"合理化的答案"
   - ✅ 喝茶、散步、吃饭——让对话在"非访谈"状态下发生
   - ✅ 每次走访带本子记**原话**（不总结、不翻译）
   - ✅ 拍照、录音（经同意）、收集实物（宣传单 / 老照片 / 他们做的手工）
3. **找"光泽 / 包浆"**：
   - 这里有什么**独特的生活方式和人的活动**？这就是城市 / 乡村的光泽 [LX1-52]
   - 哪些东西"看起来旧旧的、乱乱的，但其实很珍贵"——这就是包浆
4. **找"愿意改变的人"**：不是问"你愿不愿意"，是观察**他们自己主动在做什么小改变**

**产出**：一份田野笔记 ≥ 20 条原话引用 + ≥ 5 张现场照片 + 3 个"光泽点"列表 + 3–5 个"愿意改变的人"画像

**关键追问**：
- "如果今天你停在办公室没去现场，你的方案还会一样吗？"
- "你走访时问的问题，是不是让对方用你的话回答你？（那就是你在自证）"

---

### 【第 3 节】流程设计（Service Blueprint）· 第 1–2 周（正式工作坊开始后）

**目的**：**把愿景变成流程图**——用户在什么时候、在哪里、跟谁、做什么、感受如何？

**动作**：
1. 画一张 4 层的服务蓝图：
   - **第 1 层 用户动作**：用户在每个时刻做什么（进门 → 看到 → 被引导 → 参与 → 离开 → 回家后分享）
   - **第 2 层 前台接触点**：用户能看到/碰到的（标识 / 人员 / 空间 / 设备 / 界面）
   - **第 3 层 后台支持**：用户看不到但必须运转的（清洁 / 物料 / 系统 / 后勤）
   - **第 4 层 时间与情绪**：每个时刻用户预期花多久？情绪曲线是怎样？
2. **用人民城市三层次检查**（娄式）：
   - 1 层：硬环境（空间、设备、物料）够不够？
   - 2 层：人的活动（内容、节目、互动）有没有"内容创新"？
   - 3 层：社群（离开后会不会留下关系）会不会凝聚？
3. **做减法**：画完后，用红笔划掉你觉得"不做也行"的 20%

**产出**：完整的 4 层服务蓝图（可以用纸+便利贴，也可以用 Figma/Miro）+ 红笔划掉的"减法清单"

**关键追问**：
- "你这三层都有吗？还是只有第一层（硬环境）？如果只有第一层，这就是手术式装修，不是设计。"
- "划掉的那 20%，真的不能做吗？再划 10% 试试？"

---

### 【第 4 节】原型制作（Prototyping）· 第 3–6 周

**目的**：**做一个 70% 能跑的版本**。不求完美，求可运行、可迭代、可止损。

**动作**：
1. 原则：**低成本 + 快速原型 + 可丢弃**
   - 材料先用现成的 / 借的 / 二手的（火眼实验室就是气膜这么快 [LX2-18]）
   - 数字原型先用现成工具（Notion / Figma / 表单 / 微信群）
   - 空间用临时搭建（NICE 2035 从 200 米弄堂起步 [LX1-43]）
2. **邀请 3 类破圈方来试**：
   - 一个工程师：会不会崩？
   - 一个做小生意的：能不能持续？
   - 一个社区大妈：她会不会来？
3. **记录反馈**：
   - 哪里 work？（继续）
   - 哪里崩了？（修）
   - 哪里他们不理解？（很可能你在设计界自产自销）[LX1-32]
4. **关键约束**：原型阶段**不要**追求美观、不要追求完整、不要追求媒体曝光

**产出**：一个 70% 能跑的原型 + 3 份破圈反馈 + 1 份"下一版改什么"清单

**关键追问**：
- "你这原型如果今天崩了，你会损失多少？如果 > 你愿意承担的上限，说明不够'低成本'。"
- "这 3 个破圈方有没有一个说'我愿意继续参与'？没有的话，原型的意义到底在哪？"

---

### 【第 5 节】执行清单（Execution Checklist）· 工作坊当天

**目的**：让现场**即使出意外**也能按部就班推进。

**动作**：输出一份**执行清单**，按时间轴排列，每一项都包含：
- 时间（从几点到几点）
- 地点（具体到房间 / 角落）
- 负责人（谁在主事）
- 物料（需要什么）
- 备选方案（出意外怎么办）

**典型一天工作坊的清单模板**：

```
工作坊当天 SOP
═══════════════════════════════════════════════════
08:30  场地布置     @负责人A  |  桌椅 + 物料 + 投影 + 茶水
                               备选：投影坏了用白板
09:00  签到/破冰    @负责人B  |  名片 / 便利贴 / 破冰小活动
                               备选：来的人少于 5 → 合并圆桌
09:30  故事引入     @主持人   |  先讲故事（为什么做、为谁做）
                               备选：PPT 放不出来 → 直接讲
10:00  田野分享     @在地人   |  至少 2 位"愿意改变的人"讲
                               备选：有人临时爽约 → 主持人补
11:00  共创环节 1   @全体    |  画利益相关者地图（A0 纸 + 便利贴）
12:30  午餐         @负责人C  |  共餐，不要分组吃（社群养成）
13:30  共创环节 2   @全体    |  画服务蓝图 4 层
15:00  原型制作     @全体    |  用现成物料做 70% 原型
17:00  破圈试跑     @破圈嘉宾 |  请 3 位非设计专业的人试
18:00  反思收尾     @主持人   |  每人一句：你今天学到什么？
19:00  共餐         @全体    |  不做工作安排，纯社交
═══════════════════════════════════════════════════
```

**关键追问**：
- "你有没有预留'什么都可能出错'的 buffer？真实工作坊 30% 的意外是正常的。"
- "清单上每个环节，如果砍掉还能跑吗？砍不了的是核心，能砍的是 nice-to-have。"

---

### 【第 6 节】反思与迭代（Reflection & Iteration）· 工作坊后 1–2 周

**目的**：工作坊**不是终点是起点**。把经验沉淀为可复用的知识 + 决定下一步。

**动作**：
1. **3 × 3 反思矩阵**：
   | | 预期发生的 | 意外发生的 |
   |---|---|---|
   | **好** | 哪些如期成功？为什么？ | 哪些意外的好事？能否制度化？ |
   | **坏** | 哪些如期失败？为什么？ | 哪些意外的坏事？下次怎么避免？ |
2. **破圈检验**：
   - 除了内部人，工作坊后有没有"**外部的人主动联系**"？（媒体报道、企业投资、政府关注、居民持续参与）
   - 如果没有 → 回去重新想"你是不是在设计界自产自销"
3. **知识化**：
   - 写一份 1000–2000 字的**工作坊回顾**（不是好看的报告——是真实的反思）
   - 至少列出 3 条**下次要改的点** + 2 条**可以告诉别人的方法论沉淀**
4. **决定下一步**：
   - **继续放大**：把 200 米变成 800 米，把一条街变成整个街道（NICE 2035 路径）[LX1-43]
   - **平行复制**：在另一个社区 / 村试同样的模式
   - **止损**：如果 3–5 个关键指标都说"不行"，**诚实承认**并退出；不是所有项目都适合继续
5. **温暖收尾**：无论结果如何，办一个"反思晚餐"——把所有参与者请回来吃饭，不讨论工作，只建立关系。"社群创新"的本质是人的连接 [LX1-52]

**产出**：3×3 反思矩阵 + 工作坊回顾文章 + 下一步决定（继续 / 复制 / 止损）+ 反思晚餐邀请名单

**关键追问**：
- "如果这场工作坊一年后来回看，你觉得它最大的价值是什么？——如果答不出来，说明你没把它定位清楚。"
- "有没有人说'我愿意跟你一起做第二届'？有 → 有社群价值；没有 → 社群层可能没到。"

## 对象分型（按工作坊类型区分策略）

| 工作坊类型 | 特殊模块 | 关键陷阱 |
|---|---|---|
| **乡村振兴 / 城乡互动** | 选村三标准 + 乡村振兴四范式 + 针灸式介入 | "情怀绑架" / 外来者替代本地叙事 |
| **社区更新 / 城市微更新** | 人民城市三层次诊断 + 包浆 / 光泽识别 | 过度"洁癖" / 推倒重建 |
| **跨学科共创 / 产学研** | 中芬中心"95% 非设计生"模式 + 立体 T 型互补 | 变成单边科普 / 设计圈自嗨 |
| **AI / AIGC × 设计** | 人机协同分工表 + "AI 能做"vs"只有人能做"清单 | AI 炫技 / 伦理自动正确 |
| **K-12 设计教育工作坊** | PBL 60-40 模式 + "玩的心态" | 成人视角强塞 / 应试化 |
| **政府 / 政策 / 治理类** | 做减法清单 + 从小账到大账 + 三看方向 | 做成汇报秀 / 单向宣讲 |
| **企业内部创新** | 全流程覆盖 + 破圈方邀请 | 只有研发部门 / 没有用户 |

## 典型工作坊脚本示例（基于 NICE 2035 模板）

> 当用户问"我想做一个类似 NICE 2035 的社区创新工作坊"时，输出此模板变体：

**项目名**：[社区名] 2035 未来生活原型
**周期**：10–12 周（前 4 周准备 + 中 4 周开展 + 后 4 周迭代）
**核心隐喻**：把社区作为**创新前端**，把大学 / 机构作为**开放接口**
**选点 3 标准**：选一条 200 米 / 最被忽视的街 / 有 ≥ 3 户愿意改变的居民

| 周次 | 节 | 主要动作 | 产出 |
|---|---|---|---|
| W1–2 | 第 1 节 | 利益相关者地图 + 内外环人选 | 三环地图 + "愿意改变的 3 人"名单 |
| W3–4 | 第 2 节 | 喝茶式田野 ≥ 10 户 | 原话笔记 + 光泽点 ≥ 3 个 |
| W5 | 第 3 节 | 服务蓝图 + 减法清单 | 4 层蓝图 + 20% 划掉项 |
| W6–7 | 第 4 节 | 低成本原型（空间临时改造 / 小活动 / 数字群） | 70% 可跑原型 + 3 份破圈反馈 |
| W8 | 第 5 节 | **100 米弄堂开放日**（或等量级 1 天工作坊）| 现场记录 + 30 人参与 + 3 家媒体接触 |
| W9–10 | 第 6 节 | 3×3 反思 + 反思晚餐 + 决定下一步 | 回顾文章 + 下步计划（继续放大 200→800 米？） |
